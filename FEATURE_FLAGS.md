# Feature Flags

Features from Claude Code that are gated behind GrowthBook feature flags
(Anthropic-internal A/B system). These have been marked ⬛ descoped in
PARITY.md because conduit has no feature flag system yet.

When a feature flag system is added (e.g. local JSON config, env vars, or
a LaunchDarkly/GrowthBook-compatible client), these can be re-enabled.

---

## Implementation note

CC reads flags via:
```ts
getFeatureValue_CACHED_MAY_BE_STALE('flag_name', defaultValue)
getDynamicConfig_BLOCKS_ON_INIT<ConfigType>('config_name')
```

A minimal Go replacement could be a JSON file at `~/.claude/feature_flags.json`
with a flat map of flag names to values, loaded at startup. Each feature below
lists the flag name(s) it checks.

## Flagged features

### Session memory compaction
**Source:** `services/compact/sessionMemoryCompact.ts`
**Flags:** `tengu_session_memory` (bool, default false) + `tengu_sm_compact` (bool, default false)
**What it does:** An experimental alternative compaction strategy that uses the
  session summary (`summary.md`) generated by SessionMemory as the compaction
  output instead of asking the model to summarize. Avoids a compaction LLM call.
**Go work needed:** Port `sessionMemoryCompact.ts` (630 LOC) into a new
  `internal/compact/sessionmemory.go`, wire into `agent/loop.go` alongside the
  existing auto-compact trigger.

### Session memory
**Source:** `services/SessionMemory/`
**Flags:** `tengu_session_memory` (bool, default false)
**What it does:** Per-session `summary.md` file updated by a background sub-agent
  every N turns; injected as system context on `--continue`. Currently ✅ done
  in conduit (`internal/sessionmem/`), but in CC it's also gated by this flag.
  In conduit it is always on.

### KAIROS / assistant mode
**Source:** `assistant/`, `coordinator/coordinatorMode.ts`
**Flags:** GrowthBook `assistant` feature gate (Anthropic-internal only)
**What it does:** Proactive log-watching agent that fires autonomously when
  certain triggers are detected (build failures, test regressions, etc.).
**Go work needed:** Large scope. Requires `internal/kairos/` package with file
  watcher, trigger matchers, and coordinator integration.

### BriefTool
**Source:** `tools/BriefTool/`
**Flags:** `isKairosCronEnabled` (GrowthBook) — bundled with KAIROS
**What it does:** Tool that sends a short "brief" message to the coordinator
  when the agent has a summary to report. Used in KAIROS mode.
**Go work needed:** Simple tool, but only useful with KAIROS.

### ScheduleCronTool
**Source:** `tools/ScheduleCronTool/`
**Flags:** `isKairosCronEnabled` (GrowthBook) — bundled with KAIROS
**What it does:** Creates/manages cron schedules for KAIROS triggers.
**Go work needed:** `internal/tools/schedulecontrol/` + cron runner.

### Rate limit options dialog
**Source:** `commands/rate-limit-options/`
**Flags:** `isClaudeAISubscriber()` check + GrowthBook billing gates
**What it does:** When rate-limited, shows a dialog with upgrade / buy extra
  usage options. Integrates with billing endpoints (`extra-usage`, `upgrade`).
**Go work needed:** Billing API integration. Only relevant for Claude.ai paid
  subscriptions that support overage purchases.

### Extra usage / upgrade flow
**Source:** `commands/extra-usage/`, `commands/upgrade/`
**Flags:** `hasClaudeAiBillingAccess()` (billing-gated)
**What it does:** Allows purchasing additional usage above the subscription plan.
**Go work needed:** Would need Anthropic billing API integration.

### Feedback dialog
**Source:** `components/Feedback.tsx`, `commands/feedback/`
**Flags:** None (always available), but the endpoint is Anthropic-internal.
**What it does:** In-app feedback form that POSTs to an Anthropic endpoint.
**Go work needed:** `internal/commands/feedback.go` + POST to feedback endpoint.
  Descoped because the endpoint is internal.

### MagicDocs
**Source:** `services/MagicDocs/`
**Flags:** Anthropic-internal; not publicly documented.
**What it does:** Automated documentation generation tied to Anthropic's internal
  code intelligence pipeline.
**Go work needed:** Unknown — endpoint is internal.

## Env-var-gated features (no flag system needed)

These are disabled by an env var rather than a flag. They can be enabled today
by setting the relevant env var.

| Feature | Env var | Default |
|---------|---------|---------|
| Session memory compaction | `DISABLE_CLAUDE_CODE_SM_COMPACT=1` disables | off (also flag-gated) |
| Disable cost warnings | `DISABLE_COST_WARNINGS=1` | off |
| Disable telemetry | `DISABLE_TELEMETRY=1` | off |
| Max MCP output tokens | `MAX_MCP_OUTPUT_TOKENS=<n>` | unlimited |
| Max thinking tokens | `MAX_THINKING_TOKENS=<n>` | model default |
| MCP timeout | `MCP_TIMEOUT=<ms>` | 30000 |
| MCP tool timeout | `MCP_TOOL_TIMEOUT=<ms>` | 60000 |
| Enable tool search | `ENABLE_TOOL_SEARCH=1` | off |
| Use builtin ripgrep | `USE_BUILTIN_RIPGREP=1` | off |
